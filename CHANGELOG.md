# Changelog

Notable work on the OSRS Farming Route Optimizer. Newest first.

## Account sync: teleports/unlocks + multi-select crop grid (PR #6)
- **Account sync now fills Teleports & Other Unlocks**, not just level/quests/diaries.
  - Reliable signals from quests/diaries/Farming level (fairy rings, Lunar/Ancient
    spellbooks, ectophial, Explorer's ring, Ardougne cloak, Farming cape; Farming Guild
    45/65/85, Prif, Fossil Island, Harmony, Varlamore).
  - Item-ownership signals from a bank scan (`src/sync/teleportItemIds.js`, generated by
    `scripts/gen-teleport-ids.mjs` from a complete item dump — 112 ids / 21 families).
  - **Additive merge** (derivers emit only `true`) so manual / POH-mounted / STASHed
    teleports are never wiped. Not-derivable (planted spirit trees, Kastori site, Fire of
    Nourishment, Arceuus favour) stay manual.
- **Multi-select crop grid** replaces the per-type dropdown; `cropSelections` is now
  `{type: [cropId,...]}`. The engine (`allocateCrops`) distributes selected crops across
  patches **limited by owned seed counts** (round-robin), shows `🌱 Plant: …` per stop with
  any "out of seeds" leftover, and withdraws only what you'll plant. Graceful with no sync.
- New quest indices verified against RuneLite `Quest.java` (index = id−1, validated vs the
  9 known anchors + live data).

## Hourly auto-sync (PR #4)
- Auto-sync on open (if stale) + hourly while the tab is open; toggle in Account Sync
  (default on). Client-side only (static site). Future goal: sync even when the app is
  closed via a Cloudflare Cron Trigger + KV (see handoff / Claude memory).

## Account sync via Cloudflare Worker (PR #3)
- Group Ironman tracker (`groupiron.men`) data behind a private Cloudflare Worker proxy
  (token = Worker secret; CORS locked to the site). Pure decoders in `src/sync/`
  (skills/quests/diaries/items/derive). Bank-material-aware routing: patch types with no
  plantable owned seed are greyed/skipped. WiseOldMan fallback for Farming level by username.

## Rune label cleanup (PR #5)
- Bank "Runes" + Total Items collapse "Law rune"/"Law rune ×2" → one "Law runes".

## Deep review fixes (PR #1) + GitHub Pages deploy (PR #2)
- 63 verified findings from a multi-agent review: OSRS game-data corrections, engine bugs
  (charter ordering, profile-crash guard, inventory overflow), Farming-level input,
  session persistence, copy/export, accessibility, error boundary, React.memo, module
  split (data/engine/App), ESLint + vitest + CI, React 19 / Vite 8 upgrade (0 vulns).
- GitHub Pages auto-deploy on push to main. Live: https://nalozada.github.io/osrs_farming_route_optimizier/
